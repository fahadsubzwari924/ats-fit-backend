The project uses typescript and nestjs along with typeorm and a postgres database.

- Always follow the engineering best practices, SOLID principle when writing code.
- Aleays use the best practices of Nestjs framework.
- Always use TypeORM for database interactions.
- Always think about the scalability, readability and maintainability of the code.
- Don't use any technical term in the naming of variables, functions, classes, files or folders. Only use business terms.
- Code should be clean, well structured readable and maintainable.
- Don't create documentation files unless asked.
- Don't create interfaces, types or enums inside service or controller files, always create them in separate files inside their respective folders with respect to business context and domain.
- Always analyze the current directory structure and create the files in their appropriate places based on engineering best practices and considering modular boundaries and scability.
- Always run the build and lint after any change and looks fo errors, if found so fix those errors
- Always ask about technical solution/approach if there are multiple best options based on engineering best practices
- Don't over engineer any solution, keep it simple, easy to extend, readable,scalable and maintainable
- Always take care of performance from the beginning
